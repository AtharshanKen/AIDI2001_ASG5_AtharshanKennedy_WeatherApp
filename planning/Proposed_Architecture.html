Weather Dashboard for Outdoor Activity Days

Revised architecture focused on one deterministic flow: Python ETL -> Gold data -> Express app -> grounded answers.

Production flow
Local development and test flow

Open-Meteo API

Seasonal forecast source for the five fixed cities.

Python ETL

Extract, transform, score outing quality, and prepare app-ready data.

Bronze

Raw JSON response captured per city request date.

Silver

Cleaned detailed forecast with temp, wind, rain, and weather code.

Gold

Minimal app layer with avg temp, condition, outing score, label, and reason.

GCP Bucket

Production storage for Bronze, Silver, and Gold objects.

Express App on Vercel

Reads Gold, renders the dashboard, and assembles deterministic answers for the three fixed questions.

User Dashboard

City drop-down, 30-day table, and three predefined question buttons.

Playwright

Headed local e2e proof: select Toronto, see table, ask for great days.

Local Gold Files

Stable sample data for local development and repeatable browser testing.

OpenAI Formatting

Turns deterministic answer payloads into short user-friendly text with template fallback.

Question ownership

ETL owns daily facts and outing scoring. The Express app owns question assembly: Q1 shows the 30-day table, Q2 computes the 7-day average, and Q3 filters upcoming Great Day results.
